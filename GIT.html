GIT y sus Sentencias

Hablemos de las 4 sentencias en GIT que consideramos importantes:

Git init: Inicializa un nuevo repositorio de Git en un directorio vacío o convierte un directorio existente en un repositorio de Git. Esto crea un subdirectorio `.git` que contiene todos los archivos necesarios para el control de versiones.

Git clone: Crea una copia local de un repositorio remoto en tu máquina. Utilizado para descargar un proyecto completo de un servidor remoto, lo que te permite colaborar en el desarrollo o realizar contribuciones.

Git add: Prepara los cambios realizados en los archivos de tu proyecto para que Git los incluya en el próximo commit. Es el primer paso en el proceso de realizar un commit en Git.

Git commit: Registra los cambios preparados en el comando `git add` en la historia del repositorio. Cada commit tiene un mensaje que describe los cambios realizados y proporciona un punto de referencia en el historial de versiones.

Estas son algunas de las sentencias más fundamentales y ampliamente utilizadas en Git. Son esenciales para iniciar un repositorio, clonar un proyecto, preparar cambios y realizar commits en el control de versiones. Git ofrece muchas más sentencias y características avanzadas para un control de versiones completo y efectivo.
